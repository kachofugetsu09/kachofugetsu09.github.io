选择排序

从第一个元素到最后一个元素中选择出一个最小的元素，和第一个元素进行交换，然后从第二个元素到最后一个元素中选择出最小的，与第一个进行交换，
以此类推排列，比如213排列为123

冒泡排序

不断比较相邻的元素，将数值大的元素往后冒从而实现排序到达最后升序排列

插入排序

将第二个元素和第一个元素进行比较，如果第二个元素更小，那么第一个元素向后移动，第二个元素插入第一个元素的位置
第三个元素和第一个元素第二个元素进行比较，如果把他插入到合适的位置


数组[5, 2, 9, 3, 6]
第一次迭代：

i = 1, x = 2, j = 0，数组状态变为 [2, 5, 9, 3, 6]
第二次迭代：

i = 2, x = 9, j = 1，数组状态变为 [2, 5, 9, 3, 6]（不需要移动元素，9已在正确位置）
第三次迭代：

i = 3, x = 3, j = 2，数组状态变为 [2, 3, 5, 9, 6]（将3插入到正确位置）
第四次迭代：

i = 4, x = 6, j = 3，数组状态变为 [2, 3, 5, 6, 9]（将6插入到正确位置）
最后，数组已按升序排列：[2, 3, 5, 6, 9]。每次迭代都通过比较 x 和 a[j]，将 x 插入到已排序的子数组中的正确位置，直到整个数组都排好序为止。这就是插入排序的工作方式。

  1. 找出井字棋的获胜者
    A 和 B 在一个 3 x 3 的网格上玩井字棋。

井字棋游戏的规则如下：

玩家轮流将棋子放在空方格 (“ “) 上。
第一个玩家 A 总是用 “X” 作为棋子，而第二个玩家 B 总是用 “O” 作为棋子。
“X” 和 “O” 只能放在空方格中，而不能放在已经被占用的方格上。
只要有 3 个相同的（非空）棋子排成一条直线（行、列、对角线）时，游戏结束。
如果所有方块都放满棋子（不为空），游戏也会结束。
游戏结束后，棋子无法再进行任何移动。
给你一个数组 moves，其中每个元素是大小为 2 的另一个数组（元素分别对应网格的行和列），它按照 A 和 B 的行动顺序（先 A 后 B）记录了两人各自的棋子位置。

如果游戏存在获胜者（A 或 B），就返回该游戏的获胜者；如果游戏以平局结束，则返回 “Draw”；如果仍会有行动（游戏未结束），则返回 “Pending”。

你可以假设 moves 都 有效（遵循井字棋规则），网格最初是空的，A 将先行动。

示例 1：

输入：moves = [[0,0],[2,0],[1,1],[2,1],[2,2]]
输出：”A”
解释：”A” 获胜，他总是先走。

Q:为什么在这个计算里 要比较subel和choice

A:在这个计算中，比较subel和choice的目的是确保计算的中点不是相同的两个点。井字棋的规则是玩家必须在不同的位置放置他们的标志（通常是”X”或”O”），而不是在同一个位置多次放置。因此，在计算中点坐标之前，代码通过比较subel和choice来确保它们代表不同的步骤。

如果subel和choice相等，这意味着它们代表相同的步骤，计算的中点将会是这个相同的点，这在游戏规则下是不允许的。因此，通过检查subel和choice是否相等，代码确保只有在不同的步骤之间才会计算中点坐标。这是为了确保井字棋规则的合法性。

一种过程：在你提供的情况下，如果 lista 包含坐标 (0, 0), (1, 1), (2, 2)，那么 subel 的迭代会如下：

第一次迭代：subel 是 (0, 0)，因为它是 lista 中的第一个元素，并且与 choice 不相等。然后，程序会计算中点 (x, y)，即 (0 + 0) / 2 = 0.0 和 (0 + 0) / 2 = 0.0，然后检查 (0.0, 0.0) 是否在 lista 中。由于 (0.0, 0.0) 不在 lista 中，迭代继续。

第二次迭代：subel 是 (1, 1)，同样因为它不等于 choice。程序会计算中点 (x, y)，即 (1 + 0) / 2 = 0.5 和 (1 + 0) / 2 = 0.5，然后检查 (0.5, 0.5) 是否在 lista 中。由于 (0.5, 0.5) 不在 lista 中，迭代继续。

第三次迭代：subel 是 (2, 2)，同样因为它不等于 choice。程序会计算中点 (x, y)，即 (2 + 0) / 2 = 1.0 和 (2 + 0) / 2 = 1.0，然后检查 (1.0, 1.0) 是否在 lista 中。由于 (1.0, 1.0) 不在 lista 中，迭代结束。

Q：class Solution:
def lengthOfLastWord(self, s: str) -> int:
return len(s.strip().split(‘ ‘)[-1])
为什么能返回最后一个词的长度 是先执行了strip操作删去了前后空格 而后删除中间空格 最后从中取出最后一项吗
请问方法执行的顺序是从前往后吗
A:是的，这段代码会返回最后一个词的长度。让我解释一下代码的执行顺序：

s.strip()：首先，strip() 方法会删除字符串 s 开头和结尾的空格。这个操作会返回一个新的字符串，其中不包含开头和结尾的空格。

split(‘ ‘)：接下来，split(‘ ‘) 方法会将上一步得到的字符串按照空格分割成一个字符串列表。这个列表中的每个元素都是原字符串中的一个单词。

[-1]：最后，[-1] 表示从列表中取出最后一个元素，也就是最后一个单词。

len(…)：最后，len(…) 函数用于计算字符串的长度，这里计算的是最后一个单词的长度。

所以，代码的执行顺序是从前往后，先去除前后空格，然后按照空格分割成单词列表，最后取出列表中的最后一个单词并返回它的长度。这就是为什么这段代码能够返回最后一个词的长度。
